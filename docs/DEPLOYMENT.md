# Deployment Guide

## Prerequisites

| Requirement | Version | Purpose |
|-------------|---------|---------|
| Python | 3.11+ | Backend and AI services |
| Node.js | 18+ | Frontend |
| Ollama | Latest | Local AI inference |
| Docker | Latest (optional) | Containerized deployment |

---

## Quick Start — Docker Compose

The fastest way to get CrimeMatrix running:

```bash
git clone https://github.com/your-org/CrimeMatrix.git
cd CrimeMatrix

# Create .env file with your API keys (optional — Ollama works without keys)
cp backend/.env.example backend/.env

# Start all services
docker compose up
```

This starts:
- **Frontend** at `http://localhost:5173`
- **Backend API** at `http://localhost:8000`
- **AI Services** at `http://localhost:8002`
- **Ollama** at `http://localhost:11434`
- **Redis** at `http://localhost:6379`

Ollama will automatically download the `llama3.2:1b` model on first run.

## Manual Setup

### 1. Backend

```bash
cd backend

# Create virtual environment
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Configure environment
cp .env.example .env
# Edit .env with your settings (defaults work for local development)

# Seed database with demo data
python seed_crimes.py

# Start the server
uvicorn main:app --port 8000 --reload
```

The backend is now running at `http://localhost:8000`. API docs at `http://localhost:8000/docs`.

### 2. AI Services

```bash
cd ai-services

# Create virtual environment
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Start Ollama (in a separate terminal)
ollama serve
ollama pull llama3.2:1b

# Start the server
uvicorn main:app --port 8002 --reload
```

The AI services are now running at `http://localhost:8002`. API docs at `http://localhost:8002/docs`.

### 3. Frontend

```bash
cd frontend

# Install dependencies
npm install

# Start development server
npm run dev
```

The frontend is now running at `http://localhost:5173`.

## Environment Variables

### Backend (`backend/.env`)

| Variable | Default | Description |
|----------|---------|-------------|
| `GEMINI_API_KEY` | (empty) | Google Gemini API key (optional) |
| `OPENAI_API_KEY` | (empty) | OpenAI API key (optional) |
| `OLLAMA_BASE_URL` | `http://localhost:11434` | Ollama server URL |
| `DEFAULT_AI_PROVIDER` | `gemini` | Default AI provider |
| `REDIS_URL` | `redis://localhost:6379/0` | Redis connection URL |
| `APP_NAME` | `CrimeMatrix` | Application name |
| `APP_ENV` | `development` | Environment (development/production) |
| `APP_DEBUG` | `true` | Enable debug mode |
| `APP_HOST` | `0.0.0.0` | Server host |
| `APP_PORT` | `8000` | Server port |
| `LOG_LEVEL` | `INFO` | Logging level |

### AI Services

AI services read environment variables directly:

| Variable | Default | Description |
|----------|---------|-------------|
| `OLLAMA_BASE_URL` | `http://localhost:11434` | Ollama server URL |
| `OLLAMA_MODEL` | `llama3.2:1b` | Default Ollama model |
| `OPENAI_API_KEY` | (empty) | OpenAI API key |
| `OPENAI_MODEL` | `gpt-4o-mini` | Default OpenAI model |
| `GEMINI_API_KEY` | (empty) | Google Gemini API key |
| `GEMINI_MODEL` | `gemini-2.0-flash` | Default Gemini model |

### Frontend

| Variable | Default | Description |
|----------|---------|-------------|
| `VITE_API_URL` | `http://localhost:8001/api/v1` | Backend API URL |
| `VITE_AI_URL` | `http://localhost:8002/api/ai` | AI Services URL |

## Seeding Demo Data

The seed script populates the database with realistic Karnataka crime data:

```bash
cd backend
python seed_crimes.py
```

This creates:
- **16 Karnataka districts** (Bengaluru Urban through Ramanagara)
- **18 crime types** with severity levels (Theft, Robbery, Murder, Cybercrime, etc.)
- **40 realistic crime cases** with titles, descriptions, statuses, and priorities
- **Police stations** for major districts

## Troubleshooting

### Port already in use

```bash
# Find process using the port
lsof -i :8000  # or netstat -tlnp | grep 8000

# Kill the process
kill <PID>
```

### Ollama not responding

```bash
# Check if Ollama is running
curl http://localhost:11434/api/tags

# Restart Ollama
ollama serve
```

### Database errors

```bash
# Re-seed the database
cd backend
rm -f data/crimematrix.db
python seed_crimes.py
```

### Frontend can't connect to backend

Ensure the backend is running and the `VITE_API_URL` environment variable is correct. The frontend proxies API requests to the backend.

### AI services can't connect to Ollama

Ensure Ollama is running and the model is pulled:

```bash
ollama pull llama3.2:1b
curl http://localhost:11434/api/tags
```
